Inline tags

Headings, paragraphs and lists are block-level elements and are defined using block-level tags.
However, there exists a variety of inline tags which are not associated with a new line.

There are 2 types of inline tags:

  • Logical inline tags supply additional semantic meaning within a block.
  • Physical inline tags provide visual style.
LOGICAL Tag Enclosed content… Visual appearance
<em> …given extra emphasis Italics
<strong> …given strong emphasis Bold
<cite> …a citation or reference Italics + details in a title attribute
<code> …is program code monospace font
<abbr> and <acronym> …is a shortened form -
(requires a title attribute)
PHYSICAL <b> - Bold
<i> - Italics
<big> - Big font size
<small> - Small font size
<sub> - Subscript Text
<sup> - Superscript Text
<u> - Underline
<s> - Strikethrough

Note: "Browsers display <strong> as <b>, and <em> as <i>. However, there is a difference in the meaning of these tags: <b> and <i> defines bold and italic text, but <strong> and <em> means that the text is "important".”

Avoid deprecated physical tags as alternatives exist (CSS)!

Inline tags must be nested inside a block-level tag!
